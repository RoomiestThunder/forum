## 🗄️ **Схема базы данных SQLite**

### **Таблицы и их связи:**

1. **`users`** - Пользователи
   - `id` (PK, AUTO_INCREMENT)
   - `email` (UNIQUE, NOT NULL) 
   - `username` (NOT NULL)
   - `password` (NOT NULL, bcrypt hash)

2. **`categories`** - Категории постов
   - `id` (PK, AUTO_INCREMENT)
   - `name` (UNIQUE, NOT NULL)

3. **`posts`** - Посты
   - `id` (PK, AUTO_INCREMENT)
   - `user_id` (FK → users.id)
   - `title` (TEXT)
   - `content` (TEXT)
   - `created_at` (DATETIME)

4. **`post_categories`** - Связь многие-ко-многим
   - `post_id` (FK → posts.id)
   - `category_id` (FK → categories.id)

5. **`comments`** - Комментарии
   - `id` (PK, AUTO_INCREMENT)
   - `post_id` (FK → posts.id)
   - `user_id` (FK → users.id)
   - `content` (TEXT)
   - `created_at` (DATETIME)

6. **`post_likes`** - Лайки/дизлайки постов
   - `id` (PK, AUTO_INCREMENT)
   - `post_id` (FK → posts.id)
   - `user_id` (FK → users.id)
   - `is_like` (BOOLEAN: true=лайк, false=дизлайк)

7. **`comment_likes`** - Лайки/дизлайки комментариев
   - `id` (PK, AUTO_INCREMENT)
   - `comment_id` (FK → comments.id)
   - `user_id` (FK → users.id)
   - `is_like` (BOOLEAN)

8. **`sessions`** - Сессии пользователей
   - `id` (PK, AUTO_INCREMENT)
   - `user_id` (FK → users.id)
   - `uuid` (UNIQUE, текст UUID)
   - `expires` (DATETIME)

## 🔧 **Подробный анализ функций**

### **1. Инициализация и сервер**

#### `main()` 
```go
func main() {
    startServer()
}
```
**Назначение:** Точка входа приложения, запускает веб-сервер.

#### `startServer()`
**Назначение:** Главная функция конфигурации HTTP сервера
**Что делает:**
- Создает HTTP мультиплексер для маршрутизации
- Настраивает защиту статических файлов (только CSS и favicon)
- Блокирует доступ к `/templates` и большинству `/static` файлов  
- Подключается к SQLite базе данных
- Инициализирует таблицы БД
- Настраивает все HTTP маршруты
- Запускает сервер на порту 8080

**Маршруты:**
- `/` - главная страница, регистрация, логин, создание постов
- `/post/{id}` - детальная страница поста + добавление комментариев
- `/edit_comment`, `/delete_comment`, `/like_comment` - операции с комментариями
- `/edit_post`, `/delete_post`, `/like_post` - операции с постами
- `/static/css/*`, `/static/favicon.ico` - статические файлы (остальное заблокировано)

#### `initDB()`
**Назначение:** Инициализация базы данных
**Что делает:**
- Создает все необходимые таблицы с внешними ключами
- Добавляет базовые категории ("General", "Programming", "Offtopic")
- Создает таблицу сессий
- Обрабатывает ошибки и завершает программу при критических проблемах

---

### **2. Обработка ошибок**

#### `render400()`, `render404()`, `render405()`, `render500()`
**Назначение:** Централизованная обработка HTTP ошибок
**Что делают:**
- Устанавливают соответствующий HTTP статус код
- Отображают страницу ошибки с понятным сообщением
- Обеспечивают консистентный пользовательский опыт

---

### **3. Аутентификация и сессии**

#### `registerHandler()`
**Назначение:** Регистрация новых пользователей
**Логика:**
- **GET:** Отображает форму регистрации
- **POST:** Обрабатывает данные регистрации
  - Валидирует email (5-50 символов, формат с @)
  - Валидирует username (3-20 символов, только буквы/цифры Unicode, без пробелов)
  - Валидирует пароль (6-32 символа)
  - Проверяет уникальность email и username (case-insensitive)
  - Хеширует пароль с bcrypt
  - Создает запись в БД
  - Перенаправляет на страницу логина

#### `loginHandler()`
**Назначение:** Авторизация пользователей
**Логика:**
- **GET:** Отображает форму входа
- **POST:** Проверяет учетные данные
  - Принимает email ИЛИ username как логин
  - Ищет пользователя в БД (case-insensitive)
  - Проверяет пароль с bcrypt.CompareHashAndPassword
  - Создает UUID сессию на 24 часа
  - Удаляет старые сессии пользователя (одна сессия на пользователя)
  - Устанавливает HTTP cookie с сессией
  - Перенаправляет на главную

#### `logoutHandler()`
**Назначение:** Выход из системы
**Логика:**
- Удаляет сессию из БД
- Очищает cookie (MaxAge: -1)
- Перенаправляет на главную

#### `getCurrentUser()` и `getUserBySession()`
**Назначение:** Получение текущего пользователя из сессии
**Логика:**
- Читает cookie сессии
- Ищет сессию в БД по UUID
- Проверяет срок действия сессии
- Возвращает данные пользователя или nil

---

### **4. Посты**

#### `homeHandler()`
**Назначение:** Главная страница с постами и фильтрацией
**Мощный функционал:**
- **Пагинация:** 5 постов на страницу с навигацией
- **Фильтрация:**
  - По категориям: `/?category=1`
  - Мои посты: `/?filter=myposts` (только для авторизованных)
  - Понравившиеся: `/?filter=liked` (только для авторизованных)
- **Данные для каждого поста:**
  - Основная информация (заголовок, контент, автор, дата)
  - Счетчики лайков/дизлайков
  - Список категорий
  - Количество комментариев
- **Автоматический редирект** на предыдущую страницу если текущая пуста

#### `createPostHandler()`
**Назначение:** Создание новых постов
**Логика:**
- **Требует авторизации**
- **GET:** Показывает форму с списком категорий
- **POST:** Создает новый пост
  - Валидирует заголовок (max 20 символов Unicode)
  - Валидирует контент (обязательный)
  - Требует минимум одну категорию
  - Создает запись в posts
  - Связывает с категориями в post_categories
  - Перенаправляет на главную

#### `editPostHandler()`
**Назначение:** Редактирование постов
**Безопасность:** Только автор может редактировать
**Логика:**
- Проверяет авторство через user_id
- **GET:** Загружает текущие данные поста
- **PUT (через POST+_method):** Обновляет пост
- Валидирует данные
- Перенаправляет на главную

#### `deletePostHandler()`
**Назначение:** Удаление постов
**Безопасность:** Только автор может удалять
**Каскадное удаление:**
1. Удаляет связи категорий (post_categories)
2. Удаляет лайки поста (post_likes)  
3. Удаляет лайки комментариев (comment_likes для всех комментариев поста)
4. Удаляет комментарии (comments)
5. Удаляет сам пост (posts)

#### `likePostHandler()`
**Назначение:** Лайки/дизлайки постов
**Умная логика toggle:**
- Если лайка нет → добавляет лайк/дизлайк
- Если тот же лайк → удаляет (отменяет)
- Если противоположный → меняет на новый
- Перенаправляет обратно с сохранением параметров URL

#### `postDetailHandler()`
**Назначение:** Детальная страница поста
**Функционал:**
- **GET:** Отображает пост + все комментарии
- **POST:** Добавляет новый комментарий (только авторизованные)
- Загружает категории поста
- Загружает все комментарии с лайками/дизлайками
- Форматирует даты в удобный вид
- Валидирует комментарии (max 500 символов)

---

### **5. Комментарии**

#### `editCommentHandler()`
**Назначение:** Редактирование комментариев
**Безопасность:** Только автор может редактировать
**Логика:**
- **GET:** Показывает форму с текущим текстом
- **PUT:** Обновляет комментарий
- Валидирует длину (1-500 символов)
- Перенаправляет обратно к посту

#### `deleteCommentHandler()`
**Назначение:** Удаление комментариев
**Безопасность:** Только автор может удалять
**Каскадное удаление:**
1. Удаляет лайки комментария (comment_likes)
2. Удаляет сам комментарий (comments)

#### `likeCommentHandler()`
**Назначение:** Лайки/дизлайки комментариев
**Аналогично лайкам постов:**
- Toggle логика (добавить/отменить/изменить)
- Перенаправление обратно к посту

---

### **6. Валидация**

#### `isValidEmail()`
**Назначение:** Валидация email адресов
**Проверки:**
- Ровно один символ @
- Минимум 1 символ до @ и 3 после
- Домен содержит точку
- Домен не начинается/заканчивается точкой

#### `isValidUsername()`
**Назначение:** Валидация имен пользователей
**Проверки:**
- Длина 3-20 символов (Unicode-aware через []rune)
- Только буквы и цифры (включая Unicode)
- Запрет пробелов и табуляций

#### `isLetter()` и `isDigit()`
**Назначение:** Проверка типов символов
**Поддержка:** Latin, Cyrillic, расширенные Unicode символы

---

### **7. Шаблонизация**

#### Глобальная переменная `templates`

**Функции-помощники:**

- `dec(a)` / `inc(a)` - декремент/инкремент для пагинации
- `paginationURL()` - генерирует URLs с параметрами для пагинации

**Шаблоны:**
- `home_page.html` - главная с постами и фильтрами
- `post_detail.html` - детальная страница поста
- `create_post.html` - форма создания поста

- `login.html` - форма входа

## 🔐 **Безопасность**

### **Реализованные меры:**
1. **Хеширование паролей:** bcrypt с солью
2. **Сессии:** UUID с истечением срока (24 часа)
3. **Авторизация:** Проверка владельца при редактировании/удалении
4. **SQL инъекции:** Параметризованные запросы
5. **Статические файлы:** Ограниченный доступ (только CSS и favicon)
6. **HTTP методы:** Строгая проверка методов для каждого эндпоинта
7. **CSRF защита:** Форма-based DELETE/PUT через POST+_method

### **Контроль доступа:**
- Незарегистрированные: только просмотр постов/комментариев
- Зарегистрированные: создание постов/комментариев, лайки, фильтрация
- Авторы: редактирование/удаление своих постов/комментариев

## 🚀 **API Эндпоинты**

### **Публичные (без авторизации):**
- `GET /` - главная страница
- `GET /?category={id}` - фильтр по категории
- `GET /?page={n}` - пагинация
- `GET /post/{id}` - детальная страница поста
- `GET /login` - форма входа
- `POST /login` - авторизация
- `GET /register` - форма регистрации
- `POST /register` - создание аккаунта

### **Требуют авторизации:**
- `GET /?filter=myposts` - мои посты
- `GET /?filter=liked` - понравившиеся посты
- `GET /create_post` - форма создания поста
- `POST /create_post` - создание поста
- `GET /like_post?id={id}&like={0|1}` - лайк/дизлайк поста
- `POST /post/{id}` - добавление комментария
- `GET /like_comment?id={id}&like={0|1}&post={id}` - лайк/дизлайк комментария
- `GET /logout` - выход

### **Требуют авторства:**
- `GET /edit_post?id={id}` - форма редактирования поста
- `PUT /edit_post?id={id}` (через POST+_method) - обновление поста
- `DELETE /delete_post?id={id}` (через POST+_method) - удаление поста
- `GET /edit_comment?id={id}&post={id}` - форма редактирования комментария
- `PUT /edit_comment?id={id}&post={id}` (через POST+_method) - обновление комментария
- `DELETE /delete_comment?id={id}&post={id}` (через POST+_method) - удаление комментария

## 🐳 **Docker**

### **Dockerfile:**
```dockerfile
FROM golang:1.24-alpine

WORKDIR /app

# Установить зависимости для CGO и SQLite
RUN apk add --no-cache build-base sqlite-dev

COPY . .

# Важно: включить CGO
ENV CGO_ENABLED=1

RUN go build -o forum

EXPOSE 8080
CMD ["./forum"]
```

### **Команды запуска:**
```bash
# Сборка образа
docker build -t go-forum .

# Запуск контейнера
docker run -p 8080:8080 go-forum
```
